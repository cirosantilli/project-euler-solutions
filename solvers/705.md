# Project Euler 705 Solution - Total Inversion Count of Divided Sequences

<https://projecteuler.net/problem=705>:

* [705.py](705.py)

This solution computes

- `G(N)`: the concatenation of all primes `< N` in increasing order, with every `0` digit removed
- `F(N)`: the sum of inversion counts over **all** “divided sequences” obtained by replacing each digit by one of its divisors

All results are taken modulo `1_000_000_007`.

## Key ideas

### 1) Aggregate inversion totals
For any digit sequence `x[0..L-1]`, the inversion count is:

\[
\text{inv}(x) = \sum_{0 \le i < j < L} [x_i > x_j]
\]

Rather than generating every divided sequence, the solver keeps totals over all
generated prefixes.  Let:

- `T` be the number of divided prefixes,
- `I` be the sum of their inversion counts,
- `A[v]` be the total number of occurrences of value `v` over all prefixes.

### 2) Streaming the next digit
For a master digit `d`, the allowed values are its positive divisors in
`1..9`, and `W[d]` is the number of such divisors.

When processing a new master digit `d`, every old prefix branches into
`W[d]` new prefixes.  For each allowed appended value `v`, the new inversions
are all earlier values larger than `v`:

\[
\sum_{v\mid d}\sum_{u>v} A[u].
\]

The old inversion total and occurrence counters are multiplied by `W[d]`, and
the fresh last digit contributes `T` to each divisor value.

### 3) Chunked linear transforms
The per-digit update is linear in the state `(T, I, A[1], ..., A[9])`.  For a
base-10000 chunk, the composition has the compact form:

\[
\begin{aligned}
T' &= mT,\\
A'[v] &= mA[v] + b[v]T,\\
I' &= mI + \sum_v c[v]A[v] + eT.
\end{aligned}
\]

The solver precomputes these coefficients for every integer `0..9999`, using
the nonzero decimal digits of the chunk.  A prime below `10^8` then needs at
most two transform applications: the high base-10000 chunk and the low chunk.

### 4) Fast prime generation
`G(10^8)` is long, so primes are generated with a **segmented odd-only sieve**:

- sieve only odd numbers in each segment
- mark composites with slicing (`bytearray[start::step] = ...`) for speed
- stream primes without storing them all

### 5) Avoiding per-prime string conversions
Converting millions of primes to strings is expensive. Instead, each prime is
split into base-10000 chunks:

- precompute the non-zero decimal digits for every integer `0..9999`
- precompute the DP transform for every such chunk
- for a prime `p`, apply the transforms for `divmod(p, 10000)` in order
- zero digits naturally disappear from the transform
