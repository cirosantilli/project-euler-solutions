# Solution Outline

Let `G(c, d)` be the maximum number of tokens that can be concentrated in one square.

## 1. Reverse-process recurrence

Work in reverse: if a square has at least 2 tokens, one reverse move sends one token to each source square.

If we start with `M` tokens at one target square, write `M = 2h + r` (`r in {0,1}`).
Halting depends only on `h`, and:

- `G(c, d) = 2 * H(c, d) + 1`
- `H(c, d)` is the largest `h` for which the reverse process halts.

For reduced `(c, d)` (after dividing by `gcd(c,d)`), the carry sequence is:

`t_0 = h`, `t_n = floor((t_{n-d} + t_{n-(c+d)}) / 2)` for `n >= 1`, and `t_n = 0` for `n < 0`.

This sequence always becomes constant; halting is exactly the case where the eventual constant is `0`.

So `H(c,d)` is found by binary searching `h` with this recurrence.

## 2. Structural reduction used for speed

Define `S(n) = H(1, n)`.

For reduced `(c, d)`, we use:

- `H(c, d) = S(d + (c - 1)//2)`
- except for reduced `(c,1)` with `c in {2,3,4,5,6,8,10}`.

Those exceptions are handled explicitly.

This turns the full problem into:

1. Precompute `S(n)` up to `n = limit + (limit-1)//2`.
2. For each pair `(c,d)`, reduce by `gcd`, map to `H`, then to `G = 2H+1`.
3. Sum all values.
