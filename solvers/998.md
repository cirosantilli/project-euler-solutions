# Project Euler 998 solution summary

This solution avoids scanning all integer-sided triangles.  It generates only the triangles that can possibly have an integer minimum bounding square.

## Geometry used

For a fixed triangle, the minimum square occurs in one of two ways:

1. **Edge-aligned case**: one side of the square is parallel to a triangle side.
2. **Balanced case**: the square touches the triangle in all four side directions.

The code uses integer arithmetic for both cases, so there is no floating-point rounding issue.

## Pythagorean generation

If the square side is the integer `m`, then every relevant right-triangle component has the form

```text
m^2 + x^2 = hypotenuse^2
```

Therefore the first step is to generate, for every `m <= n`, all partners `x <= m` such that `(m, x, hypotenuse)` is a Pythagorean triple.  This is done with Euclid's formula for primitive triples and then all multiples:

```text
r^2 - s^2, 2rs, r^2 + s^2
```

The generated table is the backbone for both triangle families.

## Edge-aligned family

Place the triangle at

```text
(-x, 0), (y, 0), (0, m)
```

with `x` and `y` from the Pythagorean partner table for `m`.  The side lengths are

```text
x + y
sqrt(m^2 + x^2)
sqrt(m^2 + y^2)
```

The square side is `m` when `x + y <= m`.  It is actually minimal exactly when the balanced rotation is not smaller, which simplifies to

```text
x*y >= m*(m - x - y)
```

## Balanced family

Place the triangle at

```text
(0, 0), (m, u), (v, m)
```

where `u` and `v` are Pythagorean partners for `m`.  Two sides are automatically integral.  The third side is integral only when

```text
(m-u)^2 + (m-v)^2
```

is a square.  These rare candidates are checked with an exact minimum-square test.

## Exact candidate check

For a side-aligned candidate with base `d`, adjacent sides `e`, `f`, and doubled area `D`, the altitude foot is

```text
t = (d^2 + e^2 - f^2) / (2d)
```

The side-aligned bounding square is determined by the span of `{0, d, t}` and height `D/d`.

For a balanced candidate at a vertex with adjacent sides `p`, `q`, opposite side `r`, and doubled area `D`, write

```text
K = (p^2 + q^2 - r^2) / 2
B^2 = K^2 / (p^2 + q^2 - 2D)
```

The implementation compares this rational value to `m^2` using cross multiplication only.

## De-duplication and checks

Triangles are stored as sorted side-length triples, so congruent triangles are counted once.  The program includes asserts for the given values:

```text
T(40) = 346
T(400) = 76402
T(2000) = 3237036
```

The final line prints `T(10**6)`.
