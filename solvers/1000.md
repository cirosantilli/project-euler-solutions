# Bitwise meta-problem solution

Run:

```bash
python3 main.py
```

The script uses no external libraries. It checks every numerical example supplied in the statement with `assert` and prints only the requested meta-problem result.

## Main techniques

### 1. Max And: optimize one bit at a time

The contribution of bit `b` depends only on how many numbers carrying that bit are placed in each group. If `count` numbers in `1..n` contain the bit, the largest possible contribution is obtained by splitting those numbers as evenly as possible:

```text
(1 << b) * floor(count / 2) * ceil(count / 2)
```

These per-bit upper bounds can all be attained simultaneously: partition the integers by the parity of their population count. For any fixed bit, pairing values by a different toggled bit balances the two population-count parities, with at most one unpaired value. Therefore `max_and` sums the independent per-bit maxima.

### 2. Max Xor Sum: longest increasing walk in a weighted graph

Treat each integer as a graph vertex. The undirected edge between `x` and `y` has weight:

```text
(x * x) XOR (y * y)
```

A valid sequence is a walk whose edge weights strictly increase. Sort all edges by weight and apply dynamic programming:

```text
best[v] = largest sum of an increasing walk ending at vertex v
```

For an edge `(u, v)` of weight `w`, possible updates are `best[u] + w` into `v` and `best[v] + w` into `u`. All edges with the same weight are staged before committing updates, because equal-weight edges may not be chained. Edges are packed into integers before sorting to reduce memory usage.

### 3. Unreachable Nim: highest-XOR-bit characterization plus digit DP

Let:

```text
x = a XOR b XOR c
```

A nonzero-XOR position can be reached from a zero-XOR predecessor exactly when one pile could have been reduced from the matching pairwise XOR. It is unreachable precisely when:

```text
a >= (b XOR c)
b >= (a XOR c)
c >= (a XOR b)
```

Since `b XOR c = a XOR x`, compare each pile with itself XOR `x`. At the highest set bit of `x`, the comparison succeeds exactly when that pile has a `1`. Thus an unreachable triple has:

- even XOR parity in every bit above the highest set bit of `x`;
- the pattern `111` at that highest set bit;
- arbitrary lower bits.

A small binary digit DP counts bounded triples for every possible highest XOR bit while tracking which pile prefixes are still equal to the upper bound.

### 4. Meta recurrence: reduce after every multiplication

After computing the three starting values, the recurrence is evaluated iteratively. Each product is reduced modulo `1_000_000_007`, keeping the integers small while preserving the required result.
