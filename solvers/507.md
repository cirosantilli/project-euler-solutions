# Project Euler 507 Solution - Shortest Lattice Vector

<https://projecteuler.net/problem=507>:

* [507.py](507.py)

This repository contains a pure-Python solution (no external libraries) for **Project Euler 507**.

## Key ideas

### 1) View each `(V_n, W_n)` as a rank‑2 lattice
All lattice vectors have the form:

\[
D = kV_n + \ell W_n,\quad k,\ell \in \mathbb{Z}
\]

We want the **shortest non‑zero** vector under the Manhattan (L1) norm:

\[
\|D\|_1 = |D_x| + |D_y| + |D_z|
\]

So each `S(n)` is a shortest‑vector problem in a **2D lattice embedded in 3D**.

### 2) 2D lattice reduction, but for the L1 norm
For Euclidean norm, the classic 2D reduction is Gauss/Lagrange reduction.  
Here we need the **L1** version.

Given a current basis `(A, B)`, we perform **size reduction**:

\[
B \leftarrow B - mA
\]

where the integer `m` is chosen to minimize:

\[
\|B - mA\|_1 = |b_1 - ma_1| + |b_2 - ma_2| + |b_3 - ma_3|
\]

#### Breakpoint candidates
As a function of real `m`, this is a convex piecewise-linear function. Writing

\[
|b_i - ma_i| = |a_i|\cdot\left|m - \frac{b_i}{a_i}\right|
\]

shows that the only places where the slope can change are the coordinate ratios
`b_i/a_i` for non-zero `a_i`. A minimum over the integers is therefore found by testing
the integer points next to those breakpoints:

- `floor(b_i/a_i)`
- `floor(b_i/a_i) + 1`

The current vector `B` is the `m = 0` candidate, so in 3 dimensions each reduction step needs
at most six fresh L1 evaluations. This is a little more direct than sorting the ratios to find
the weighted median, and it is much friendlier to PyPy on this problem.

### 3) Stop only at a truly reduced pair
The reduction loop always keeps the shorter vector first. It replaces

\[
B \leftarrow B - mA
\]

only when the chosen `m` strictly lowers `||B||_1`. When no tested integer shear improves
`B`, the reduced pair already accounts for `B + A` and `B - A` through the `m = -1` and
`m = 1` candidates. Since `B` is kept no shorter than `A`, the answer is the smaller
non-zero norm of the two reduced basis vectors.

### 4) Streaming Tribonacci residues
The residues satisfy:

\[
r_i = (r_{i-1}+r_{i-2}+r_{i-3}) \bmod 10^7
\]

We **stream** them in a single loop, buffering only the current 12-value block used to build
`V_n` and `W_n`. Because each sum is `< 3·10^7`, the modulus is implemented with at most two
subtractions instead of `%`, which is faster.

The code asserts the test values given in the problem statement:

- `S(1) = 32`
- `sum_{n=1..10} S(n) = 130762273722`

Running `python3 main.py` prints the required sum for `n = 1..20_000_000`.
