# Solution Summary

This program computes the requested value for the polynomial divisibility problem without using external libraries.

## Main ideas

For a prime `p`, the polynomial

```text
f_p(x) = 1 + x + ... + x^(p-1)
```

is the cyclotomic polynomial for primitive `p`-th roots of unity. If `z` is such a root, then `f_p` divides `g_s` exactly when `g_s(z) = 0`.

After reducing every divisor exponent modulo `p`, this means the coefficients of the residues must all be equal. A valid `s` cannot be divisible by `p`, so the condition becomes:

```text
the divisors of s must represent every nonzero residue modulo p exactly once.
```

Thus `tau(s) = p - 1`.

## Cyclic group formulation

The nonzero residues modulo `p` form a cyclic group of order `m = p - 1`. Choose a primitive root `r`, and replace every residue by its discrete logarithm base `r`. A prime factor `q^e` in `s` contributes the interval

```text
0, log(q), 2 log(q), ..., e log(q)  (mod m)
```

The divisors of `s` cover all nonzero residues exactly once when these intervals form a direct tiling of the cyclic group.

## Dynamic programming

The code grows the covered subgroup size `h`, where `h` is always a divisor of `m`.

If the current subgroup has size `h`, the quotient has size `M = m / h`. Expanding by a factor `L` uses a prime `q` whose coset has exact order `L` in that quotient. This contributes `q^(L-1)` to `s` and moves the state from `h` to `hL`.

The dynamic program stores the smallest exact integer that reaches each subgroup size. The target state is `h = m`.

## Fast transition lookup

For a prime `q`, only

```text
gcd(log_r(q), m)
```

is needed to determine all quotient orders. The program scans ordinary primes once for each `p` and records the least prime producing each possible gcd class. This makes transition costs quick to look up during the dynamic program.
