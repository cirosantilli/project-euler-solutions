# Main techniques used

The problem's equation is the asymmetric one

\[
\frac{a}{b+c} + \frac{b}{a+c} + \frac{c}{a+c} = 4,
\]

so the last two terms have the same denominator.

## 1. Reduce to a quadratic in `b + c`
Set

\[
t = b+c.
\]

Then the equation becomes

\[
\frac{a}{t} + \frac{t}{a+c} = 4,
\]

which is equivalent to

\[
t^2 - 4(a+c)t + a(a+c) = 0.
\]

Therefore the discriminant must be a perfect square:

\[
(a+c)(3a+4c) = \square.
\]

## 2. Use coprime square-factor splitting
Let

\[
g = \gcd(a+c, c), \qquad a+c = gu, \qquad c = gv,
\]

with `gcd(u, v) = 1`.
Then

\[
(a+c)(3a+4c) = g^2 u(3u+v).
\]

Since

\[
\gcd(u, 3u+v) = \gcd(u, v) = 1,
\]

and their product is a square, each factor must be a square:

\[
u = m^2, \qquad 3u+v = n^2.
\]

That gives the full parameterization

\[
a = g(4m^2-n^2),
\]
\[
c = g(n^2-3m^2),
\]
\[
b = g(5m^2-n^2 \pm mn).
\]

## 3. Positivity constraints
For positive integers `a, b, c` we need

\[
3m^2 < n^2 < 4m^2.
\]

For the `+` branch, that already guarantees `b > 0`.
For the `-` branch, we also need

\[
5m^2 - n^2 - mn > 0.
\]

## 4. Count all multiples from primitive solutions
Every solution is a positive multiple of a unique primitive one (`g = 1`).
If a primitive solution has sum `s`, then all of its valid multiples contribute

\[
s + 2s + \cdots + \left\lfloor \frac{L}{s} \right\rfloor s
= s \cdot \frac{M(M+1)}{2},
\]

where `L = 10^7` and `M = floor(L / s)`.

So the program only enumerates primitive solutions, computes their primitive sums,
and adds the arithmetic-series contribution of all allowed multiples.

## 5. Speed improvement for the minus branch
For the `-` branch, define

\[
k = 2m - n.
\]

Then

\[
s = a+b+c = k(5m-k),
\]

and the positivity conditions reduce to a narrow interval for `m` in terms of `k`.
This makes the search comfortably fast within pure Python and without external
libraries.

## 6. Validation
The program includes small exact cross-checks against a brute-force solver for small
limits, plus a couple of direct parameterization checks.
