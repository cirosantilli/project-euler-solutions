# Project Euler 631 Solution - Constrained Permutations

<https://projecteuler.net/problem=631>:

* [631.py](631.py)

The task is to compute `f(10^18, 40)` modulo `1,000,000,007`, where `f(n,m)` counts permutations of every size from `0` through `n` that avoid the pattern `1243` and have at most `m` inversions.

## Key idea

Instead of constructing a permutation by choosing the value of its last element, the faster implementation inserts the new maximum value at each step. If the current length becomes `length`, the insertion creates some number

```text
0 <= inversions < length
```

of new inversions. This makes the inversion budget a local transition rather than a property that has to be rediscovered from the whole partial permutation.

The history needed to enforce the pattern restriction is summarized by a triple:

```text
(remaining, lower, threshold)
```

`remaining` is the unused inversion budget. `lower` is the smallest insertion contribution still allowed by the 132-type structural restriction. `threshold` separates the two update cases induced by the 21-type restriction.

For each state and each legal insertion contribution

```text
lower <= inversions < min(remaining + 1, length)
```

the transition is:

```text
inversions < threshold:
    (remaining - inversions, inversions + 1, threshold + 1)

inversions >= threshold:
    (remaining - inversions, lower, inversions)
```

This is the important speedup over the older port: the state now tracks only the future-relevant insertion boundaries, so the DP needs far fewer layers and less pattern bookkeeping.

## Large n

Once `length >= m + 2`, the `length` cap in `min(remaining + 1, length)` is no longer active because `remaining <= m`. From that point onward, the same finite-state transition is repeated every step, and the exact-length count has stabilized.

So the implementation explicitly computes layers only through `m + 2`, sums those layers into the running prefix total, and then appends the linear tail:

```text
(n - (m + 2)) * stable_layer_count
```

For `m = 40`, this means only 42 non-empty layers are needed even though `n = 10^18`.

## Complexity

There are `O(m^3)` possible triples and each state can try up to `O(m)` insertion contributions. Only `O(m)` layers are built before stabilization, giving a straightforward `O(m^5)` worst-case bound with `O(m^3)` memory. The constants are small for `m = 40`.
