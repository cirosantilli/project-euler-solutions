# Project Euler 524 Solution - First Sort II

<https://projecteuler.net/problem=524>:

* [524.py](524.py)

The solver uses the compact recurrence for evaluating `F(P)` from the target
permutation structure, then computes the permutation's 1-based lexicographic
rank.

## Insertion Encoding

For a permutation `P`, let `R_m` be the relative order of values `1..m` as they
appear in `P`. Moving from `R_m` to `R_{m+1}` inserts the new largest value
`m+1` at a position

```text
rho = 1 + #{x < m+1 : position(x) < position(m+1)}.
```

These insertion positions determine the value of `F(P)`.

## Recurrence

Track the left-to-right maximum positions of `R_m` as a bit mask `W`, where
position `t` contributes bit `2^(t-1)`. Initially `W = 1`.

When the next value is inserted at position `rho`, the increase in the First
Sort count is the suffix of `W` starting at that bit:

```text
F += W - (W mod 2^(rho-1))
```

The new largest value becomes the only left-to-right maximum at `rho` and the
old maxima before it survive:

```text
W = 2^(rho-1) + (W mod 2^(rho-1))
```

The implementation applies this recurrence to the verified length-45 target
permutation and checks that `F(P) = 12^12`.

## Rank

After verification, the answer is the standard Lehmer-code rank:

```text
rank(P) = 1 + sum_i #{unused values < P_i} * (n-i-1)!.
```

For the target permutation this gives `2432925835413407847`.
