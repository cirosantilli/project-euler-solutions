# Project Euler 483 Solution - Repeated Permutation

<https://projecteuler.net/problem=483>:

* [483.py](483.py)

The goal is

\[
g(n)=\mathbb{E}_{\sigma\in S_n}\left[\operatorname{ord}(\sigma)^2\right],
\]

where the order of a permutation is the least common multiple of its cycle
lengths.

## Cycle-type sum

If a permutation has \(m_c\) cycles of length \(c\), then

\[
\sum_c c m_c=n,
\qquad
\#\text{permutations}=\frac{n!}{\prod_c c^{m_c}m_c!},
\]

and

\[
\operatorname{ord}(\sigma)=\operatorname{lcm}\{c:m_c>0\}.
\]

After dividing by \(n!\), the expectation is a sum over cycle multiplicities:

\[
g(n)=
\sum_{\sum c m_c=n}
\frac{\operatorname{lcm}\{c:m_c>0\}^2}
{\prod_c c^{m_c}m_c!}.
\]

## DP by largest prime factor

The solver keeps sparse histograms `dp[used]`, mapping a tracked lcm value to the
accumulated cycle-type weight for total used length `used`.

Fixed points are initialized first:

\[
dp[t][1]=\frac1{t!}.
\]

Then cycle lengths \(c\ge2\) are processed in descending blocks of their largest
prime factor. For one length \(c\), adding \(m\ge1\) cycles multiplies the weight
by

\[
\frac1{c^m m!}
\]

and changes the key to \(\operatorname{lcm}(\ell,c)\).

After the whole block for a prime \(p\) is complete, no later cycle length can be
divisible by \(p\). Therefore every key

\[
\ell=p^a u,\qquad p\nmid u
\]

can be replaced by \(u\), while the stored weight is multiplied by \(p^{2a}\).
This absorbs the completed contribution to the final squared order and merges
many lcm states.

After all prime blocks are processed, the answer is

\[
g(n)=\sum_\ell \ell^2\,dp[n][\ell].
\]

The output is formatted with 10 significant digits, matching the problem
examples. The code checks \(g(3)\), \(g(5)\), and \(g(20)\).
