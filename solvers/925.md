# Project Euler 925 Solution - Larger Digit Permutation III

<https://projecteuler.net/problem=925>:

* [925.py](925.py)

The target is the sum of `B(n^2)` for `1 <= n <= 10^16`, modulo
`1_000_000_007`.  The endpoint `n=10^16` contributes zero, so the computation is
equivalent to summing over `1 <= n < 10^16`.

## Main ideas

Write

```text
B(x) = x + delta(x)
```

where `delta(x)` is `B(x)-x` if a larger digit permutation exists, and `-x`
when no larger permutation exists.  The sum of the `x = n^2` part is the closed
form square sum.  The hard part is the correction sum.

The correction is found from right to left.  A root is represented as

```text
n = (q * 10^width + suffix) * 10^trailing_zeros
```

where `suffix` ends in a nonzero digit and `q` is the still-unseen higher
prefix.  All roots sharing this state have the same low digits of the square:

```text
n^2 == suffix^2 * 10^(2 * trailing_zeros)
       (mod 10^(width + 2 * trailing_zeros)).
```

The recursion prepends one digit to `suffix`.  The known square suffix is kept
non-increasing from left to right.  When the newly exposed square digit is
smaller than the previous leftmost known digit, the rightmost ascent used by the
next-permutation algorithm is fixed inside the known suffix.  Higher prefix
digits will not affect the correction, so the whole block of remaining prefixes
is added at once.

There is one padding-zero case: if the root prefix is empty and the square of
the visible suffix has fewer digits than the fixed width, the apparent leading
zero is not a real digit.  That one completion uses `delta(actual_suffix_square)`;
all other completions use a representative with a real digit to the left of the
same decisive suffix.

## Checks

The implementation keeps the statement checks:

- `B(245) = 254`
- `B(542) = 0`
- `T(10) = 270`
- `T(100) = 335316`

## Complexity

Only suffix states that stay non-increasing continue recursing.  Most branches
collapse after a few digits, and each collapsed branch needs only one or two
next-permutation evaluations on at most `2k` digits.  Memory use is just the
small power tables plus recursion depth.
