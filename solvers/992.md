# Solution summary

We count frog journeys by fixing the final stone first.

## 1. Directed edge counts

For a fixed endpoint `t`, let `right[i]` be the number of traversals from stone `i` to stone `i+1`.
Because the frog visits stone `i` exactly `k+i` times for `0 <= i < n`, these counts satisfy a very small recurrence:

- `right[0] = k - [t = 0]`
- `right[1] = 2 - [t = 1]`
- `right[i] = 1 + right[i-2] - [t = i]` for `i >= 2`

So for any chosen endpoint, all directed edge counts are determined in linear time.

## 2. Count local departure orders

Once the edge counts are known, the whole walk is determined by the order of left/right departures at each internal stone.

For an internal stone `v`:

- if `v < t`, the last departure from `v` must go right,
- if `v > t`, the last departure from `v` must go left,
- if `v = t`, there is no forced last departure.

That makes the number of valid local orderings a binomial coefficient:

- `C(out(v)-1, right[v]-1)` when `v < t`
- `C(out(v), right[v])` when `v = t`
- `C(out(v)-1, right[v])` when `v > t`

Multiplying these over all internal stones gives the number of walks ending at `t`.
Summing over `t = 0..n` gives `J(n, k)`.

## 3. Modular arithmetic

The modulus `987898789` is prime, so binomial coefficients can be computed efficiently with:

- factorials modulo `MOD`
- inverse factorials modulo `MOD`
- Fermat inverses: `x^(MOD-2) mod MOD`

This keeps every binomial query `O(1)` after precomputation.

## 4. Complexity

For one value of `k`:

- There are `n+1` possible endpoints.
- Each endpoint is processed in `O(n)` time.

So the total cost is `O(n^2)` per `k`, with `O(max(k)+n)` preprocessing for factorials.
