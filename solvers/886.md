# Project Euler 886 Solution - Coprime Permutations

<https://projecteuler.net/problem=886>:

* [886.py](886.py)

For `n = 2m`, the numbers `2..2m` contain one more even number than odd
number.  Since two even numbers are never coprime, every valid permutation has
the forced pattern

\[
E-O-E-\cdots-O-E.
\]

Adding `1` between the two even endpoints turns this path into a Hamiltonian
cycle in the balanced bipartite graph with odd side

\[
U=\{1,3,\ldots,2m-1\}
\]

and even side

\[
V=\{2,4,\ldots,2m\},
\]

where an edge means coprimality.

## Inclusion-Exclusion

Let `B` be the `m x m` bipartite adjacency matrix.  With the row for `1`
distinguished, the count is evaluated as

\[
\sum_{k=0}^{m-1} \sum_{|I|=|J|=k}
(-1)^k \det(B_{I,J})^2\operatorname{perm}(B_{\bar I,\bar J})^2
\pmod {83456729}.
\]

The determinant part supplies the signs that cancel disconnected cycle covers;
the squared permanent counts ordered pairs of perfect matchings in the
complementary block.

## Grouping

Rows with identical neighborhoods are grouped, and the same is done for
columns.  A determinant block can use at most one representative from each
equal row or column class; choosing that representative contributes the class
size as a multiplicity factor.

For the complementary permanent, repeated columns are handled with grouped
Ryser summation.  If a column class has size `s`, the Ryser subset only needs
to know how many `t` of those columns were selected, with multiplicity
`\binom{s}{t}`.  Rows are grouped by their support against the remaining column
classes, so the product of row sums is raised to the row-class multiplicity.

The implementation asserts `P(4) = 2` and `P(10) = 576`, then prints
`P(34) mod 83,456,729`.
