# Project Euler 675 Solution - $2^{\omega(n)}$

<https://projecteuler.net/problem=675>:

* [675.py](675.py)

For

\[
S(n)=\sum_{d\mid n}2^{\omega(d)},
\]

we need

\[
F(N)=\sum_{i=2}^{N}S(i!) \pmod {1\,000\,000\,087}
\]

at `N = 10^7`.

## Product Formula

If

\[
n=\prod p^{e_p},
\]

then each prime contributes `1` for exponent `0` in a divisor and `2` for each
positive exponent. Therefore

\[
S(n)=\prod_p (2e_p+1).
\]

For factorials, let `E_p(i) = v_p(i!)`. Then

\[
S(i!)=\prod_p (2E_p(i)+1).
\]

## Incremental Update

When moving from `(i-1)!` to `i!`, only primes dividing `i` change. If
`p^c || i` and the old factorial exponent is `e`, the local product factor is
replaced by

\[
2e+1 \longrightarrow 2(e+c)+1.
\]

So the running value is multiplied by

\[
\frac{2(e+c)+1}{2e+1}
\]

for each prime factor of `i`.

## Implementation

The code builds a smallest-prime-factor table up to `N`, then factors each `i`
by repeated SPF lookup. It also precomputes modular inverses for every possible
odd local factor `2e+1`; the largest exponent occurs for prime `2`, so
`2*v_2(N!)+1` is a safe bound.

The loop keeps:

* `exponents[p] = E_p(i!)`
* `running = S(i!) mod MOD`
* `total = F(i) mod MOD`

No factorial value and no divisor list is ever constructed.

## Checks

The solver verifies `F(5) = 96` and compares the optimized update with a small
direct factorization version for `N = 50`.
