# Project Euler 35 Solution - Circular Primes

<https://projecteuler.net/problem=35>:

* [35.py](35.py)
* [35.lean](35.lean)

A number is a **circular prime** if every rotation of its digits is prime (e.g., 197 → 197, 971, 719).

## Key ideas

1. **Prime lookup via sieve**
   - Build a fast primality table using the Sieve of Eratosthenes.
   - For safety with arbitrary limits, sieve up to `10^d` where `d = number of digits of (limit-1)`, because rotating digits preserves digit count and stays below `10^d`.

2. **Digit pruning**
   - For any multi-digit circular prime, all digits must be from `{1,3,7,9}`.
   - If a number contains `0,2,4,5,6,8`, some rotation ends with an even digit or 5 ⇒ composite.
   - Single-digit primes `{2,3,5,7}` are handled separately.

3. **Rotation checking**
   - Generate all rotations (implemented with arithmetic shifting) and check each rotation in O(1) using the sieve table.

4. **Avoid double work**
   - Maintain a `processed` array for numbers `< limit` to avoid re-checking different rotations of the same digit-cycle.

## Result
The program computes the number of circular primes below one million and prints:

- **55**
