# Project Euler 988 Solution - Non-attacking Frogs

<https://projecteuler.net/problem=988>:

* [988.py](988.py)

# Solution summary

## Key idea
For coprime `a` and `b`, a frog at `m` attacks a frog at `n` exactly when `n - m`
is representable as

`u*a + v*b` with `u, v >= 0`.

So every frog position in a valid configuration must be a **gap** of the numerical
semigroup generated by `a` and `b`.

## Re-encoding the gaps
Every positive gap has a unique form

`ab - ai - bj`

with `i, j >= 1` and `ai + bj < ab`.

That turns the problem into a finite set of lattice points inside a Ferrers shape.
The point `(i, j)` carries the frog position `ab - ai - bj`.

## When do two frogs attack?
For two gaps

- `x = ab - ai - bj`
- `y = ab - ak - bℓ`

the difference `y - x` is representable iff `(i, j)` and `(k, ℓ)` are comparable
coordinatewise.

Therefore a non-attacking configuration is exactly an **antichain** in this poset.

## From antichains to monotone heights
Antichains in a Ferrers poset are equivalent to order ideals, and an order ideal is
encoded by a non-increasing sequence of column heights:

- column `i` can have height from `0` to `h[i]`
- heights must satisfy `t[1] >= t[2] >= ...`

A maximal cell appears whenever the height drops:

- if `t[i] > t[i+1]`, then `(i, t[i])` belongs to the antichain
- its label is `ab - ai - b*t[i]`

So the required sum can be accumulated while scanning columns left to right.

## Dynamic programming state
For each processed prefix of columns and each current height, store:

- how many ideals end with that height
- the total sum of already finalized antichain labels

When the height decreases from one column to the next, the top cell of the previous
column becomes a maximal element and its label is added.

## Complexity
If we swap so that `a <= b`, then the diagram height is at most `a - 1` and the width
is `b - 1`.

The DP runs in polynomial time and is easily fast enough for `(19, 53)`.
