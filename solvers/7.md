## Approach

We need the 10,001st prime.

### Key ideas
- Use the standard upper bound for the `n`th prime (valid for `n >= 6`):
  \[
  p_n < n(\ln n + \ln\ln n)
  \]
  This gives a good initial sieve limit, and we can double the limit if it turns out to be too small.
- Use the Sieve of Eratosthenes, optimized to store only odd numbers:
  - Index `i` represents the number `2*i + 1`.
  - Mark multiples starting from `p*p`.

### Complexity
- Time: `O(L log log L)` for sieve limit `L`
- Space: `O(L)` but halved by storing only odds

The computed result is printed at the end.
