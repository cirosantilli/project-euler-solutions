# Project Euler 1001 Solution - Connections I

<https://projecteuler.net/problem=1001>:

* [1001.py](1001.py)

The array is a chord diagram. Two values cannot both be retained exactly when
their occurrences alternate, so the requested connectivity number is the
number of independent sets in the chord-intersection graph.

## Implicit crossing components

The crossing graph can have quadratically many edges, so the program never
constructs it.

Chords are processed by increasing left endpoint. For every existing connected
component, a meldable min-heap stores its still-active right endpoints. A new
chord crosses a component exactly when that component's minimum active right
endpoint lies strictly before the new chord's right endpoint.

All such components are merged with the new chord. Each heap entry expires
once and each successful merge reduces the number of components, giving
`O(n log n)` time and `O(n)` storage for the sweep.

Independent-set counts multiply across crossing components.

## Counting one component

Many chord diagrams, including any diagram admitting a line that separates the
two endpoints of every chord, are permutation diagrams. A sliding window finds
such a separating cut. After rotating there, order the chords by their first
endpoints and record their positions among the second endpoints.

A selected set is non-crossing exactly when those second positions form a
decreasing subsequence. A Fenwick tree counts all decreasing subsequences in
`O(k log k)` time and `O(k)` memory.

For a general component, the fallback is the standard interval recurrence:

```text
F(i,j) = F(i+1,j)
       + F(i+1,mate(i)-1) * F(mate(i)+1,j)
```

when both ends of the chord at `i` lie in `[i,j]`; otherwise only the first term
is present. The implementation recomputes one right-boundary row at a time.
This costs `O(k^2)` time but only `O(k)` memory, avoiding a quadratic table.

All arithmetic is reduced modulo `1_003_443_221`.
