# Project Euler 810 Solution - XOR-Primes

<https://projecteuler.net/problem=810>:

* [810.py](810.py)

Read the binary expansion of an integer as a polynomial over `F_2`.  Under that
encoding, XOR is polynomial addition and carryless multiplication is polynomial
multiplication.  An XOR-prime is exactly an irreducible polynomial code.

The only even irreducible code is `2`, representing `x`.  Every other even code
has zero constant term and is divisible by `x`, so the sieve only stores odd
codes after counting `2`.

## Polynomial Sieve

When an odd irreducible code `p` is found, any later odd composite having `p` as
a factor is

\[
p \otimes q
\]

where `q` is an odd monic cofactor.  If `p` has degree `d`, the cofactor degree
starts at `d`, mirroring the ordinary sieve rule of starting at `p^2`; smaller
cofactors would already have been handled by a smaller irreducible factor.

For a fixed cofactor degree `j`, the top bit and constant bit of `q` are fixed.
The internal bits are traversed in Gray-code order.  Consecutive cofactors then
differ by exactly one monomial `x^r`, so the product changes by only

\[
p \otimes x^r,
\]

which is just `p << r`.  Thus each marked composite is reached from the previous
one with one XOR and one shift.

Before allocating the sieve, the code uses the standard irreducible-polynomial
count

\[
\frac1n\sum_{d\mid n}\mu(d)2^{n/d}
\]

to choose a binary length guaranteed to contain the requested rank.
