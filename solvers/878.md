# Project Euler 878 Solution - XOR-Equation B

<https://projecteuler.net/problem=878>:

* [878.py](878.py)

## Main ideas

Read each integer as a polynomial over `GF(2)`, with XOR as addition and
carryless multiplication as polynomial multiplication.  With `A` and `B` for
the polynomials of `a` and `b`, the expression in the problem is

```text
A^2 + tAB + B^2.
```

The transformation

```text
(a, b) -> (b, a ^ (b << 1))
```

preserves that quadratic form.  Its inverse is

```text
(a, b) -> (b ^ (a << 1), a).
```

So valid pairs are grouped into orbits with a fixed invariant value.

For a nonzero ordered pair, the pair is the first ordered point on its orbit
exactly when its inverse step is not ordered:

```text
b ^ (a << 1) > a
```

This gives a direct canonical-seed test.  The special pair `(0, 0)` is a fixed
orbit and is counted once.

If `m` has `d` bits, every seed with invariant at most `m` lies in the finite
box

```text
0 <= a <= b < 2^ceil((d + 1) / 2).
```

The solver scans only this seed box.  For each candidate it computes the
carryless quadratic form, keeps only canonical seeds with value at most `m`,
and then walks the orbit forward until `b > N`.  Along that walk the invariant
does not need to be checked again.

## Checks

The code keeps the statement checks:

- `7 XOR-product 3 = 9`
- `(a, b) = (3, 6)` has invariant `5`
- `G(1000, 100) = 398`

It also checks the small orbit example `G(5, 1) = 4`.

## Complexity

Let `S = 2^ceil((bit_length(m) + 1) / 2)`.  The seed scan costs `O(S^2)`, and
the forward walks add one step per counted orbit point up to `N`.  The
implementation keeps only the square table for the seed box and otherwise uses
constant working state.
