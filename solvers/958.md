# Project Euler 958 Solution - Euclid's Labour

<https://projecteuler.net/problem=958>:

* [958.py](958.py)

The subtraction-only Euclidean algorithm can be viewed as a path in the
Stern-Brocot tree.  Reversing a subtraction step replaces a pair by one of its
parents in the addition tree.  The task is to find, for a fixed `n`, the
smallest positive `m` among pairs `(n,m)` that occur at the minimum possible
depth.

## Meet in the Middle

The fast solver does not grow all paths from `(2,1)` until the first component
equals `n`.  Instead, for a candidate number of steps it splits the path near the
middle.

At a midpoint basis `(a,b)`, any completion to first component `n` can be written
as

\[
n=a\alpha+b\beta.
\]

The search carries both the basis and the coefficient pair
`(alpha, beta)`.  Coefficients are normalized by adding multiples of the lattice
vector `(b,-a)`, which preserves `a*alpha + b*beta`, until the relevant
representatives are nonnegative.

## Midpoint Checks

At the split depth, only coefficient vectors near the reduced representative can
lead to a short opposite half.  The solver uses the squared norm

\[
a^2+b^2
\]

and the cross product to reduce the coefficient vector into a fundamental
parallelogram, then checks the two neighboring representatives.  For odd total
step counts, the midpoint lies halfway through an edge, so the code performs the
same test in doubled coordinates and then enforces the required parity.

The remaining half path is simulated directly for only the small number of
steps left after the split.  The candidate answer is derived from the final
linear form and its modular inverse modulo `n`; the minimum among the symmetric
choices is kept.

The implementation verifies:

* `f(7) = 2`
* `f(89) = 34`
* `f(8191) = 1856`
