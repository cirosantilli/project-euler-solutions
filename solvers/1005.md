For every prime index and remaining sum, the solver counts how many increasing
prime lists can be formed using only later primes.  This is the usual
distinct-subset sum dynamic program, but with arbitrary-size integer counts
because only the rank is needed.

The problem's median rule is the lower median, so the required zero-based rank
is `(count - 1) // 2`.  To unrank it in lexicographic order, try the next prime
candidates in increasing order.  All lists with a fixed next prime form one
contiguous block, whose size is already available in the suffix DP table; skip
whole blocks until the rank falls inside one of them.
