# Project Euler 983 Solution - Consonant Circle Crossing

<https://projecteuler.net/problem=983>:

* [983.py](983.py)

The solver uses the same constructive family that was validated in the previous rollout:

- Pick `k` vectors from opposite lattice-point pairs on the circle `x^2 + y^2 = m`.
- Build circle centers as all even subset sums of those vectors.
- This gives exactly `2^(k-1)` circles.
- For the target (`n >= 500`), we use `k = 10`, so we search for a perfect `512`-circle set.

For each candidate radius-square `m`, the code:

1. Uses the two-squares factorization rule to count antipodal lattice-vector
   pairs before enumerating the circle.
2. Enumerates opposite pairs of lattice vectors on the circle.
3. Searches `k` chosen pairs and orientations by DFS, with one global-sign
   symmetry removed.
4. Prunes a branch as soon as a signed sum of four chosen directions is already
   a displacement between two radius vectors.
5. Builds the `512` centers only for surviving complete branches.
6. Runs a fast harmony-point count with early exit.
7. If the fast count matches `512`, runs strict validation:
   - no tangencies,
   - exactly `512` unique harmony points,
   - graph connectivity.

The two statement checks are asserted directly by search:

- `R(2)^2 = 1` via `k = 2`
- `R(4)^2 = 5` via `k = 3`

To keep runtime practical in Python/PyPy for `k = 10`, the search uses the same
narrowed candidate family from the rollout (`p in {k, k+2}` and unit-coordinate
presence among circle lattice points), then applies the displacement pruning
above and validates any hit strictly.
