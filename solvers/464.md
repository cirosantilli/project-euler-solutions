# Project Euler 464 Solution - Möbius Function and Intervals

<https://projecteuler.net/problem=464>:

* [464.py](464.py)

For an interval `I`, let `P` be the number of entries with `mu(k) = 1` and `N`
the number with `mu(k) = -1`.  The interval is counted when

\[
99P \le 100N,\qquad 99N \le 100P.
\]

The solver counts the complement.  The two bad cases are disjoint:

\[
99P - 100N > 0,\qquad 99N - 100P > 0.
\]

So

\[
C(n)=\frac{n(n+1)}2-\#A-\#B.
\]

After computing all Mobius values with a linear sieve, the two bad families are
converted into prefix-sum pair counts.  Define

\[
a(k)=
\begin{cases}
99,&\mu(k)=1,\\
-100,&\mu(k)=-1,\\
0,&\mu(k)=0,
\end{cases}
\quad
b(k)=
\begin{cases}
-100,&\mu(k)=1,\\
99,&\mu(k)=-1,\\
0,&\mu(k)=0.
\end{cases}
\]

For prefixes `A_j = sum a(k)` and `B_j = sum b(k)`, an interval belongs to the
first bad family exactly when `A_r > A_{l-1}`, and similarly for `B`.

The implementation first scans the prefix walks to find their minimum and
maximum values.  It then scans again with two dense Fenwick trees, counting for
each current prefix how many earlier prefixes are strictly smaller.  Those two
counts are subtracted from the total number of intervals.

The program verifies:

* `C(10) = 13`
* `C(500) = 16676`
* `C(10000) = 20155319`

and prints `C(20000000)`.
