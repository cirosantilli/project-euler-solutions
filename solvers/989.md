# Solution Summary

Let

- `G(n)` be the number of solutions to `x^2 ≡ x + 1 (mod n)`.
- `Q(a, b) = a^2 - ab - b^2`.

The solution uses four facts.

1. `G(n)` is multiplicative.
   For prime powers:
   - `G(2^e) = 0`
   - `G(5) = 1`, `G(5^e) = 0` for `e >= 2`
   - for odd `p != 5`, `G(p^e) = 2` if `p ≡ 1, 4 (mod 5)`, else `0`

2. Primitive reduced representations by `Q` count the same objects.
   There is a bijection
   `G(n) = #{(a,b): a >= 2b > 0, gcd(a,b)=1, Q(a,b)=n}`.

3. Fibonacci weights are handled with Binet’s formula modulo `10^9+9`.
   Since `5` has a square root modulo `10^9+9`, we can work with
   `phi = (1 + sqrt(5)) / 2` and `psi = (1 - sqrt(5)) / 2` in the finite field and write
   `F_n = (phi^n - psi^n) / sqrt(5)`.

4. The coprimality condition is removed by Möbius inversion.
   If
   `A_w(L) = sum w^{Q(a,b)}` over reduced pairs without `gcd(a,b)=1`,
   then
   `P_w(L) = sum_{g<=sqrt(L)} mu(g) * A_{w^{g^2}}(floor(L / g^2))`.

## Fast Evaluation of `A_w(L)`

Using

- `u = 2a - b`
- `v = b`
- `4Q(a,b) = u^2 - 5v^2`

and splitting by parity, `A_w(L)` becomes two families:

- `m^2 - 5t^2`
- `m(m+1) - 5t(t+1) - 1`

with sliding ranges in `m`.

For each family, the program keeps a moving window sum of:

- `w^{m^2}` for the square family
- `w^{m(m+1)}` for the triangular family

Each term enters the window once and leaves once, so one call to `A_w(L)` costs `O(sqrt(L))`.

After Möbius inversion, the total complexity is `O(sqrt(L) log L)` and the memory usage is `O(sqrt(L))` only for the Möbius table.

## Validation

`main.py` includes asserts for:

- direct brute-force values of `G(n)` on small `n`
- agreement between `G(n)`, the factorization formula, and quadratic-form counting
- small brute-force checks for the non-primitive window sum
- brute-force Fibonacci sums on small limits
- the given checkpoint

`sum_{n=1}^{1000} F_n G(n) ≡ 190950976 (mod 10^9+9)`.
