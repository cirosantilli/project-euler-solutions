# Project Euler 615 Solution - Prime Factor Count

<https://projecteuler.net/problem=615>:

* [615.py](615.py)

Let `Omega(n)` be the number of prime factors of `n`, counted with
multiplicity. The problem asks for the millionth smallest number with
`Omega(n) >= 1,000,000`, modulo `123454321`.

## Working Threshold

For a smaller threshold `c`, search only below the barrier `3^c` and count
numbers with `Omega(n) >= c`. Once at least one million such numbers exist, the
millionth value `a_c` satisfies `a_c <= 3^c`.

That inequality lets us lift to the next threshold by doubling:

```text
a_(c+1) = 2 * a_c
```

Any odd number with at least `c+1` prime factors is at least `3^(c+1)`, which is
too large to appear before `2*a_c`. So every earlier level-`c+1` candidate is
even and maps back to a level-`c` candidate by division by `2`.

Repeating this gives the final answer:

```text
a_1000000 = a_c * 2^(1000000-c).
```

## DFS Enumeration

The search enumerates products of primes in nondecreasing order. This is unique
because it is just the sorted prime factorization of each integer.

At a state with product `n`, `u` used factors, and next allowed prime `p`, the
smallest possible completion to threshold `c` is

```text
n * p^(c-u).
```

If this already exceeds `3^c`, no larger next prime can work either, so the
branch stops. The code performs the same check with logarithms.

For this input, the first successful working threshold is `c = 29`; after
sorting the collected candidates, the millionth value is lifted by modular
exponentiation.

## Check

For rank `5`, the search succeeds at `c = 3` with value `20`. Doubling twice
gives `80`, matching the statement's small example for threshold `5`.
