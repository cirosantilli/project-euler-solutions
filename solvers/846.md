# Project Euler 846 Solution - Magic Bracelets

<https://projecteuler.net/problem=846>:

* [846.py](846.py)

Adjacent labels `u,v` are compatible when `uv = x^2 + 1`.  Instead of building
the full compatibility graph and enumerating cycles, the solver represents each
admissible label as a primitive Gaussian-integer vector.

## Vectors

The only useful labels are:

* `1`
* `2`
* `p^e`
* `2p^e`

where `p == 1 mod 4`.  For each such prime, find one representation

\[
p=a^2+b^2.
\]

Powers of `a + bi` give primitive vectors for `p^e`; multiplying by `1+i`
gives the doubled family.  Each vector is canonicalized with nonnegative
coordinates sorted as `x >= y`.

If labels `m` and `n` have primitive vectors `(u,v)` and `(s,t)`, then

\[
mn=(us+vt)^2+(ut-vs)^2.
\]

So compatibility is guaranteed when `|ut - vs| = 1`.

## Parent Edges

For a label with vector `(x,y)`, a Bezout relation

\[
\alpha x+\beta y=1
\]

produces two smaller vectors:

\[
(|\beta|,|\alpha|),\qquad (x-|\beta|,y-|\alpha|).
\]

Their norms are the two parent labels.  They are compatible with each other and
with the original label.  Thus the compatibility graph can be oriented from
larger labels to smaller parent labels; every label has at most two outgoing
edges.

## Counting Bracelets

Every undirected cycle has a unique maximum label.  The solver sweeps labels in
descending order and stores, for each lower edge, open descending path fragments:

* the number of fragments;
* the sum of their interior label totals.

When the sweep reaches a real edge, all stored fragments on that edge close into
bracelets.  Then the trivial fragment for the real edge is seeded.  If the
current label has two parents, fragments from both parent edges are paired and
forwarded to the lower edge between the parents.

This dynamic program counts each bracelet exactly once, at its maximum label.
The implementation checks `F(20)=258` and `F(100)=538768`.
