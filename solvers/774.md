# Project Euler 774 Solution - Conjunctive Sequences

<https://projecteuler.net/problem=774>:

* [774.py](774.py)

We count sequences `a_1..a_N` with `0 <= a_i <= B` and
`a_i & a_{i+1} > 0`.

## Main ideas

The dynamic program counts a segment together with boundary obligations:

```text
D(length, bound, left, right)
```

`left` constrains the first value and `right` constrains the last value.  A
boundary is one of:

- no obligation,
- must be odd,
- must share a `1` bit with a marked value `v`.

The answer is `D(N, B, none, none)`.

Strip the lowest bit.  If an endpoint is even, the obligation shifts right:
`[v] -> [v//2]`; an odd-only obligation becomes impossible.  If an endpoint is
odd, an odd marked value is already satisfied by the low bit, while an even
marked value shifts to `[v//2]`.

For odd `B`, both even and odd values reduce to the common range
`0..(B-1)//2`.  Patterns with no adjacent odd-odd pair still need their higher
bits checked, and the number of such parity patterns with fixed endpoint
parities is Fibonacci.  If the first adjacent odd-odd pair occurs at a cut, that
edge is already satisfied by the low bit, so the sequence splits into an
independent prefix and suffix.

For even `B`, the top value `B` is exceptional: it is an even value with no
representative in the common reduced range.  The solver counts sequences below
`B`, then adds sequences by the first position where `B` appears.  The prefix is
restricted to values `< B`, and the suffix keeps the original bound.

All states are memoized modulo `998244353`.

## Checks

The implementation asserts the statement values:

## Complexity

Each distinct `(length, bound, left, right)` state is evaluated once.  A state
does `O(length)` split work, and the bound roughly halves through the odd-bound
recurrence, so the memo table remains small for `N = 123`.
