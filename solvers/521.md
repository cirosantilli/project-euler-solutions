# Project Euler 521 Solution - Smallest Prime Factor

<https://projecteuler.net/problem=521>:

* [521.py](521.py)

We need

\[
S(N)=\sum_{i=2}^{N}\operatorname{smpf}(i)\pmod {10^9},
\]

for `N = 10^12`.

## Filtered Tables

Before processing a prime `p`, keep the numbers that have not been removed by
smaller prime factors.  For each stored bound `x`, the solver tracks:

- `C(x)`: how many surviving values are in `2..x`,
- `T(x)`: their sum modulo `10^9`.

Initially these are just `x - 1` and `2 + 3 + ... + x`.

When `p` is prime, the composites whose smallest prime factor is `p` are

\[
p m,\qquad m\le \left\lfloor N/p\right\rfloor,
\]

where `m` also survived all smaller-prime filtering and `m >= p`.  Their count
is therefore

\[
C(\lfloor N/p\rfloor)-C(p-1),
\]

so `p` times that count is added to the answer.  The same values are then
removed from every stored `C(x)` and `T(x)` using the recurrence

\[
C(x)\leftarrow C(x)-\bigl(C(\lfloor x/p\rfloor)-C(p-1)\bigr),
\]

\[
T(x)\leftarrow T(x)-p\bigl(T(\lfloor x/p\rfloor)-T(p-1)\bigr).
\]

After every prime up to `sqrt(N)` is processed, all composites have been
removed.  The remaining `T(N)` is the sum of primes up to `N`, which is added to
the composite contribution.

## Compression

Only `O(sqrt(N))` bounds are needed: the small bounds `1..sqrt(N)` and the
large bounds `floor(N/d)` for `1 <= d <= sqrt(N)`.  The implementation stores
separate small and large tables and chooses between them when a quotient is
looked up.

The hot tables are plain Python lists.  On PyPy they are faster than
`array.array` for this update-heavy sieve.

The statement checks `smpf(91)=7`, `smpf(45)=3`, and `S(100)=1257` are asserted
in `521.py`.
