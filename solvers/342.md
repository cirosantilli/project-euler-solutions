# Project Euler 342 Solution - The Totient of a Square Is a Cube

<https://projecteuler.net/problem=342>:

* [342.py](342.py)

We need the sum of all `n < 10^10` for which `phi(n^2)` is a perfect cube.
Writing

```text
n = product p^e_p
```

gives

```text
phi(n^2) = product p^(2e_p - 1) * (p - 1).
```

So every prime exponent in this product must be `0 mod 3`.

## Descending Prime Search

The solver processes primes from large to small. This makes the cube condition
local: every prime divisor of `p - 1` is smaller than `p`, so after all primes
larger than `q` have been decided, the accumulated contribution to the exponent
of `q` is final.

For each prime `q`, let `c` be this pending exponent modulo `3`.

* If `q` is excluded from `n`, this is valid only when `c = 0`.
* If `q` is included with exponent `e`, we need
  `c + (2e - 1) = 0 mod 3`.

The smallest valid exponents are therefore:

```text
c = 0 -> e = 2
c = 1 -> e = 3
c = 2 -> e = 1
```

Residue `0` makes the prime optional. Residues `1` and `2` force the prime into
the current base solution.

## Precomputation

Only primes up to `floor(sqrt(10^10 - 1))` can start a new optional branch,
because the smallest optional exponent is `2`. The solver builds a smallest
prime factor table to that limit, lists the primes, and pre-factors each
`p - 1`, keeping only exponents modulo `3`.

When the DFS includes a prime `p`, it adds those stored `p - 1` residues to the
pending residue array for smaller primes. The updates are undone after the
recursive branch returns.

## Whole Families

Once a minimal base solution is found, any included exponent may be increased by
a multiple of `3`. This preserves the cube condition, because `2(e + 3) - 1`
differs from `2e - 1` by `6`.

For each minimal base `b`, the solver recursively sums all multipliers

```text
product p^(3t_p)
```

that keep `b * multiplier < 10^10`, then adds the whole family at once.

## Check

The implementation keeps the problem statement check that

```text
phi(50^2) = 1000 = 10^3.
```

The final answer is `5943040885644`.
