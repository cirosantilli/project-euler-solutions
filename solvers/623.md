# Project Euler 623 Solution - Lambda Count

<https://projecteuler.net/problem=623>:

* [623.py](623.py)

Count exact-size lambda terms after quotienting by alpha-equivalence.  A
variable occurrence is determined only by which enclosing binder it references,
so in a context with `d` available binders there are `d` variable choices.

Let `A_d(s)` be the number of terms of exact size `s` in such a context.  The
size rules give

\[
A_d(s)=d\mathbf{1}_{s=1}+A_{d+1}(s-5)+
       \sum_{i=1}^{s-3}A_d(i)A_d(s-2-i).
\]

The final answer is

\[
\Lambda(n)=\sum_{s=1}^{n} A_0(s).
\]

Only depths with `5d + 1 <= n` can contribute, and at depth `d` only sizes up
to `n - 5d` are needed.  The solver processes depths downward from
`floor((n - 1) / 5)` to `0`, retaining only the current depth and the next
deeper one.

For a fixed depth, the application term is maintained as a running ordered
convolution

\[
C(k)=\sum_{i+j=k}A_d(i)A_d(j).
\]

After `A_d(s)` is computed, the solver updates all future convolution entries
`C(s + u)`.  Off-diagonal size pairs are doubled because `(left, right)` and
`(right, left)` are distinct, while the diagonal contribution is added once.

The depth-zero exact counts are prefix-summed modulo `1,000,000,007`.  The
implementation checks the statement values:

* `Lambda(6) = 1`
* `Lambda(9) = 2`
* `Lambda(15) = 20`
* `Lambda(35) = 3166438`

The program prints:

\[
\Lambda(2000) \bmod 1{,}000{,}000{,}007 = 3679796.
\]
