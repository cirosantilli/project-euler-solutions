# Project Euler 344 Solution - Silver Dollar Game

<https://projecteuler.net/problem=344>:

* [344.py](344.py)

The solver computes `W(1_000_000, 100) mod 1000036000099`.

## Gap model

With `m = c + 1` coins, a placement is represented by the nonnegative gaps

```text
g0, g1, ..., gm
```

before, between, and after the coins. Their sum is `N = n - m`. For `c = 100`, `m = 101` is odd, and the ordinary silver-dollar losing condition is an xor-zero condition on the alternating active gaps:

```text
g0 xor g2 xor ... xor g100 = 0
```

The remaining gaps are passive slack that only affects the total sum.

## Losing positions

When `m` is odd, the pocketing rule splits losing positions into two families:

```text
L0 = Count(N, active=m//2+1, passive=m-(m//2+1)+1)
L1 = Count(N+1, active=m//2+1, passive=...)
     - Count(N+1, active=m//2, passive=...)
```

`L0` covers the silver dollar on the second coin. `L1` covers every coin farther right; the subtraction removes the cases where the shifted first active gap is zero. The final answer is

```text
m * C(n, m) - (L0 + (m - 2) * L1)
```

## Fast count

The previous Python port built full distributions over all possible active-gap sums up to about one million. The faster version uses the bit-column DP from the reference explanation.

For one binary column, suppose `x` active gaps and `y` passive gaps contain a `1`. The xor condition requires `x` to be even, so the number of ways to have `s = x + y` ones in the column is

```text
ways[s] = sum C(active, x) * C(passive, y)
          over even x with x + y = s
```

To count gap vectors with arithmetic sum `S`, process the bits of `S` from low to high. A DP state is just the carry entering the current bit. If a column has `s` ones, then it is legal when

```text
(carry + s) % 2 == bit(S)
```

and the next carry is

```text
(carry + s - bit(S)) // 2
```

This counts the xor-zero gap vectors directly in `O(m^2 log n)` time instead of scanning every possible sum.

## Modulus

The modulus factors as

```text
1000036000099 = 1000003 * 1000033
```

The code evaluates the formula modulo each prime, computes the small-`k` binomial coefficient multiplicatively, and combines the two residues with CRT.
