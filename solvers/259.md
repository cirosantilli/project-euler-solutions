# Project Euler 259 Solution - Reachable Numbers

<https://projecteuler.net/problem=259>:

* [259.py](259.py)

The digits `1` through `9` must be used in order. Between adjacent digits we may
concatenate or use a binary operation, and parentheses may be placed arbitrarily.

## Key ideas

Treat each of the eight gaps between consecutive digits as a small code. A gap can
be one of the four arithmetic operations, or it can be concatenation. There are
only \(5^8\) such gap patterns. Once concatenations are resolved, a pattern gives
a short list of integer tokens and a fixed list of arithmetic operators between
them.

For one fixed token/operator list, parentheses are handled by an interval dynamic
program. Let \(R(l,r)\) be the exact rational values obtainable from the token
interval \(l..r\). The base interval contains the token itself, and a larger
interval is formed by choosing the final binary operation position:

\[
R(l,r)=\bigcup_{l\le s<r} R(l,s)\operatorname{ op}_s R(s+1,r).
\]

Here \(\operatorname{op}_s\) is fixed by the gap pattern. Division by zero is
skipped. Each interval stores a set, so repeated values from different
parenthesizations collapse immediately.

Fractions are stored as reduced `(numerator, denominator)` pairs with positive
denominators. This keeps all arithmetic exact and avoids losing integer results
that pass through fractional intermediate values.

At the end of each pattern, pairs with denominator `1` and positive numerator are
inserted into one global set. The requested answer is the sum of that final set,
so integers reached by several different expressions are counted once.

## Check

The code checks that `42` is reachable, matching the example expression from the
problem statement.
