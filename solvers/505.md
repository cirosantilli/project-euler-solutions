# Project Euler 505 Solution - Bidirectional Recurrence

<https://projecteuler.net/problem=505>:

* [505.py](505.py)

The recurrence defines `x(k)` from its parent and grandparent:

\[
x(2k)=3x(k)+2x(\lfloor k/2\rfloor),\qquad
x(2k+1)=2x(k)+3x(\lfloor k/2\rfloor)
\pmod {2^{60}}.
\]

For `y_n`, internal nodes apply

\[
y_n(k)=2^{60}-1-\max(y_n(2k),y_n(2k+1)).
\]

Flipping the value on alternating depths turns this into a plain alternating
min/max game.  The important speed point is how the uneven boundary `k >= n`
is handled.

## Frontier Blocks

Let `h = floor(log2(2n - 1))` and `base = 2^h`.  The leaves of the complete
height-`h` tree correspond to indices beginning at `base`, while the true
boundary is at `2n`.

The solver partitions the frontier into maximal dyadic blocks:

* a block wholly before `2n` is an ordinary `x` minimax subtree;
* a block wholly after `2n` is already past the boundary and is evaluated with
  the complement transform;
* mixed blocks are split until they become uniform.

This avoids repeatedly testing the boundary inside a full depth-40 search.  For
each block, `x(k)` and `x(floor(k/2))` are computed by following the binary
digits of `k`; the uniform subtree is then evaluated with alpha-beta pruning and
child ordering.

## Folding

Once all frontier blocks have values, they are folded back to the root.  A node
at odd remaining depth takes `max`; a node at even remaining depth takes `min`.
The final flip depends on the height parity.

The implementation checks:

* `A(4) = 8`
* `A(10) = 2^60 - 34`
* `A(1000) = 101881`
