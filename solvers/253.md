# Project Euler 253 Solution - Tidying Up A

<https://projecteuler.net/problem=253>:

* [253.py](253.py)

We view the process as repeatedly deleting one occupied cell from a row. The occupied cells are always a disjoint union of contiguous segments, and only the multiset of segment lengths matters for the future.

## State

A state is a sorted tuple of current segment lengths, for example `(1, 3, 3)`. Absolute positions and the left-to-right order are irrelevant: deleting cells never reconnects separated segments, and equal segment lengths have identical future behavior.

The recursive state also carries `best`, the largest number of segments seen so far.

## Transitions

For one segment of length `L`:

- If `L = 1`, deleting its only cell removes that segment.
- If an endpoint is deleted, the segment becomes one segment of length `L-1`; there are two endpoint choices.
- If an interior cell is deleted, the segment splits into two positive lengths `a` and `b` with `a+b=L-1`.

Equal segment lengths are grouped, so each transition is multiplied by the number of segments of that length. Identical next states are accumulated into one transition weight.

## Expectation

Let `E(state, best)` be the expected final maximum segment count from the compressed state. If `R` cells remain, every remaining cell is equally likely to be deleted next, so

\[
E(S,M)=\frac{1}{R}\sum_{S'} w(S,S')\,E(S',\max(M,|S'|)).
\]

The empty state returns `best`. Memoization makes the recurrence finite and exact over the compressed Markov states.

The computation starts from `(40,)` with `best = 1`. The included `N=10` check verifies the known mean `385643/113400`, and the final value is printed rounded to six decimals.
