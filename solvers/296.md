# Project Euler 296 Solution - Angular Bisector and Tangent

<https://projecteuler.net/problem=296>:

* [296.py](296.py)

Let

\[
a=BC,\qquad b=CA,\qquad c=AB,
\]

with `a <= b <= c`. The triangle must also satisfy `c < a + b` and
`a + b + c <= N`.

## Geometry reduction

Using the angle-bisector setup and the tangent-chord relation, the required
length simplifies to

\[
BE = \frac{ac}{a+b}.
\]

Thus `BE` is integral exactly when

\[
a+b \mid ac.
\]

## Arithmetic count

For a fixed pair `(a, b)`, the possible third sides are

\[
b \le c \le c_{\max},\qquad
c_{\max}=\min(a+b-1,\ N-a-b).
\]

Let `g = gcd(a, b)`. Since

\[
\gcd(a,a+b)=g,
\]

the divisibility condition is equivalent to saying that `c` is a multiple of

\[
d=\frac{a+b}{g}.
\]

So instead of scanning every `c`, the code counts multiples of `d` in the valid
interval:

\[
\left\lfloor\frac{c_{\max}}{d}\right\rfloor
-
\left\lfloor\frac{b-1}{d}\right\rfloor.
\]

This removes the direct loop over `c`.

## Grouping by the reduced pair

Looping over every `(a, b)` pair is still large for `N = 100000`, so the solver
groups pairs with the same reduced ratio. Write:

\[
a=gx,\qquad b=g(q-x),\qquad \gcd(x,q)=1,\qquad x\le q/2.
\]

Then `a+b = gq`, and the divisibility condition says that:

\[
c=kq.
\]

The perimeter bound is:

\[
(g+k)q\le N.
\]

The condition `b <= c < a+b` is clearer after writing `h = g-k`:

\[
h\ge 1,\qquad h\ge 2g-\left\lfloor\frac{N}{q}\right\rfloor,\qquad
h\le \left\lfloor\frac{gx}{q}\right\rfloor.
\]

So for fixed `(q, g, x)`, the number of valid `k` values is:

\[
\max\left(0,\ \left\lfloor\frac{gx}{q}\right\rfloor-h_{\min}+1\right).
\]

## Coprime floor sums

For each `(q, g)`, the solver must sum the previous expression over all
`x <= q/2` coprime to `q`. The lower cutoff is monotone, so this becomes a
difference of two sums:

\[
\sum_{\substack{x\le X\\(x,q)=1}}\left\lfloor\frac{gx}{q}\right\rfloor
\quad\text{and}\quad
\#\{x\le X:(x,q)=1\}.
\]

Both are computed by Mobius inversion over the divisors of `q`. The floor part
uses the standard Euclidean `floor_sum` routine for:

\[
\sum_{i=0}^{n-1}\left\lfloor\frac{ai}{m}\right\rfloor.
\]

This changes the main work from billions of `gcd(a,b)` calls to roughly
\(\sum_{q\le N/3} N/q\) floor-sum blocks.

## Checks

For small limits, the optimized count is compared with a direct brute-force
enumeration of all valid triangles. It is also compared with the simpler
pair-loop formula at `N = 1000`.
