# Project Euler 376 Solution - Nontransitive Sets of Dice

<https://projecteuler.net/problem=376>:

* [376.py](376.py)

For two six-sided dice `X,Y`, let `W_XY` be the number of ordered face pairs
with `x > y`.  Since there are `36` comparisons, `X` beats `Y` exactly when
`W_XY >= 19`.

The solver counts labeled triples with the fixed cycle

\[
B>A,\qquad C>B,\qquad A>C,
\]

then divides by `3` for cyclic rotations.

## Value Sweep

For each value `v = 1..N`, store how many faces of that value are assigned to
each die:

\[
a_v,b_v,c_v.
\]

The DP state contains:

* used face counts `(u_A,u_B,u_C)`;
* capped win counts `(W_BA,W_CB,W_AC)`, where `19` means "already won".

When assigning the current value, a new `B` face beats all earlier `A` faces,
but not equal current faces.  Therefore the transition increments are

\[
\Delta W_{BA}=b_vu_A,\qquad
\Delta W_{CB}=c_vu_B,\qquad
\Delta W_{AC}=a_vu_C.
\]

## Pruning

After a transition, the solver checks an optimistic upper bound.  For example,
if `B` has `6-u_B` faces still unassigned, those faces can contribute at most
`6(6-u_B)` more wins against `A`.  So a necessary condition is

\[
W_{BA}+6(6-u_B)\ge 19,
\]

and similarly for the other two directed comparisons.  States failing any of
these bounds are discarded immediately.

States are packed into integers and stored sparsely in a dictionary.  The
implementation verifies `N=7 -> 9780` and prints the result for `N=30`.
