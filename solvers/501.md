# Project Euler 501 Solution - Eight Divisors

<https://projecteuler.net/problem=501>:

* [501.py](501.py)

We want the count of integers `n ≤ 10^12` that have **exactly 8 divisors**.

---

## 1) Divisor-count factorization patterns

If the prime factorization of `n` is:

\[
n = \prod p_i^{a_i}
\]

then the divisor count is:

\[
d(n) = \prod (a_i + 1)
\]

We need:

\[
d(n) = 8
\]

The only ways to write `8` as a product of integers `> 1` are:

- `8 = 8` → `n = p^7`
- `8 = 4×2` → `n = p^3 q` (distinct primes)
- `8 = 2×2×2` → `n = p q r` (three distinct primes)

So we count:

1. **p⁷ ≤ N**
2. **p³q ≤ N, p≠q**
3. **pqr ≤ N, distinct primes**

---

## 2) Counting with the prime counting function π(x)

To count primes under large bounds (up to about 1e11), we need the **prime counting function**:

\[
\pi(x) = \#\{p \le x : p \text{ prime}\}
\]

For this problem all prime-count queries are tied to one fixed limit `N`, so the
solver builds a dedicated lookup table instead of calling a recursive prime counter.

---

## 3) Fixed-limit prime counting table

Let `v = floor(sqrt(N))`. The solver sieves all primes up to `v`, then builds two
tables:

- `small[x] = π(x)` for `x ≤ v`
- `large[d] = π(floor(N / d))` for `1 ≤ d ≤ v`

Both tables start as counts of integers greater than 1. Processing each prime `p`
removes numbers whose least remaining prime factor is `p`, using the same inclusion
step as a combinatorial sieve. After this pass, every query needed by the counting
formulas is O(1):

- use `small[x]` when `x ≤ v`
- use `large[N // x]` when `x > v` and `x` has the form `floor(N / d)`

---

## 4) Counting each divisor-form

### A) p⁷
Count primes `p ≤ N^(1/7)` → `π(N^(1/7))`.

### B) p³q
The solver first counts all `p³q` with `q` prime, including the temporary case
`q = p`:

\[
\sum_{p \le N^{1/3}} \pi\!\left(\left\lfloor\frac{N}{p^3}\right\rfloor\right)
\]

Then it subtracts the invalid diagonal choices `q = p`, which occur exactly for
`p^4 ≤ N`, so the correction is `π(N^(1/4))`.

### C) p q r
Loop through `p < q` with constraint `p*q*q ≤ N`.
For each pair:

- `r ≤ N/(p*q)` and `r > q`
- count = `π(N/(p*q)) − π(q)`

The largest part of the runtime is this prime-pair loop, but each iteration only
does arithmetic and one table lookup.
