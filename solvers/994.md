# Counting Triangles

This solution computes `T(1234 * 10^8, 2345 * 10^8)` modulo `1_000_000_007` without hard-coding the final numeric result.

## Main ideas

Every drawn segment can be represented by a grid point `(i, j)`, meaning the segment from `(i, 1)` to `(j, 2)`.

Two such segments meet when their grid points are not in strictly increasing order in both coordinates. A triangle is obtained from a triple of segments whose three pairwise intersections are distinct. The count is therefore split into:

1. triples that pairwise meet and are not immediately degenerate at a shared endpoint;
2. minus triples of segments that all pass through the same interior point.

## Candidate triples

Sort a triple by its lower endpoint coordinate.

There are two useful cases:

- exactly two segments share a lower endpoint;
- all three lower endpoints are distinct, and the upper endpoint coordinates are weakly decreasing.

This gives the modular candidate count used in `nonconcurrent_candidate_count()`:

```text
m(m-1)n(n-1)(n+1)/6 + C(m,3) * (C(n+2,3) - n)
```

The `- n` removes the case where all three upper endpoints are identical.

## Concurrent triples

Three segments are concurrent at an interior point exactly when their corresponding grid points are collinear with negative slope.

For two extreme grid points separated by `(a, -b)`, the number of interior lattice points on the segment between them is:

```text
gcd(a, b) - 1
```

So the concurrent correction is:

```text
sum_{1 <= a < m, 1 <= b < n} (gcd(a,b)-1)(m-a)(n-b)
```

Using the identity

```text
gcd(a,b) = sum_{d | gcd(a,b)} phi(d)
```

this becomes a summation over Euler's totient function:

```text
sum_d phi(d) A_m(d) A_n(d) - C(m,2)C(n,2)
```

where

```text
A_x(d) = sum_{k=1..floor((x-1)/d)} (x-kd)
```

The code groups ranges of `d` for which both floor values stay constant, turning `A_m(d)A_n(d)` into a quadratic polynomial in `d` over each range.

## Fast totient prefix sums

The grouped formula needs prefix sums of:

```text
sum phi(d), sum d phi(d), sum d^2 phi(d)
```

The implementation sieves these up to a configurable limit, then uses a Du Jiao-style summatory recursion for larger values. For `k = 0, 1, 2`, define:

```text
F_k(n) = sum_{d <= n} d^k phi(d)
```

Since `(d^k phi(d)) * (d^k) = d^(k+1)` under Dirichlet convolution, the recursion is:

```text
F_k(n) = P_{k+1}(n) - sum_{b=2..n} b^k F_k(floor(n/b))
```

The range of `b` is grouped by equal `floor(n/b)`, and results are memoized.

## Running

```bash
python3 main.py
```

The default sieve limit is `10_000_000`. It can be tuned for the available machine:

```bash
SIEVE_LIMIT=5000000 python3 main.py
```

A larger limit uses more memory but reduces recursive summation work.
