# Project Euler 1002 Solution - Connections II

<https://projecteuler.net/problem=1002>:

* [1002.py](1002.py)

Make one graph vertex for each value and join two vertices when their chord
endpoints alternate. Connections drawn on the same side must not cross, so an
above/below assignment is exactly a two-coloring of this circle graph.

The input is promised to be bipartite-connectable. Every connected component
has two possible color orientations, and its best contribution is therefore
the larger of its two color-class sizes.

## Avoiding the quadratic crossing graph

The crossing graph may contain quadratically many edges. The solution finds a
parity spanning forest without constructing those edges.

Process chords by increasing left endpoint. An earlier chord crosses the new
chord exactly when it is still active and its right endpoint is smaller than
the new right endpoint.

For each current connected component, a meldable leftist heap stores all active
right endpoints. Its minimum determines whether that component has any
crossing neighbour of the new chord:

```text
new left < component minimum right < new right
```

If so, one crossing edge is enough to merge the two components and impose
opposite colors. The heaps are melded and the sweep continues looking for
other components.

A disjoint-set union structure stores parity to the component root and the two
color-class sizes. Each successful merge reduces the number of components,
and each right endpoint is removed from a heap only once.

The resulting complexity is:

```text
Time:  O(n log n)
Space: O(n)
```

This remains linear in memory even when the implicit crossing graph is dense.
